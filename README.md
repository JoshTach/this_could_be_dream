# GamePulse (static)

A tiny **from-scratch** website that shows:

- **Most recent update / patch note entry** for each “popular” game (via the game’s Steam News feed when available)
- **Recent gaming news** aggregated from multiple RSS feeds
- A **search box** to quickly find games

No build tools required — it’s plain **HTML/CSS/JS**.

---

## Quick start (detailed steps)

Follow these in order if you’re not sure where to begin.

### Step 1: Open a terminal in the project folder

- In Cursor (or VS Code), open the folder that contains `index.html`, `app.js`, and `server.js`.
- Open the integrated terminal: **Terminal → New Terminal** (or `` Ctrl+` `` / `` Cmd+` ``).
- Confirm you’re in the project folder (you should see `index.html` if you run `dir` on Windows or `ls` on Mac/Linux).

### Step 2: Start the local server

- In that terminal, type:
  ```bash
  node server.js
  ```
- Press Enter.
- You should see something like:
  ```text
  GamePulse dev server: http://localhost:3000
    /api/steam-news is available (no 404s or CORS for Steam).
  ```
- Leave this terminal open and running. Don’t close it while you’re using the site.

### Step 3: Open the site in your browser

- Open your browser (Chrome, Edge, Firefox, etc.).
- In the address bar, go to: **http://localhost:3000**
- Press Enter.
- You should see the GamePulse page with game cards and news. If game updates don’t load at first, wait a few seconds and refresh once.

### Step 4 (optional): Connect Supabase so login and visits work

Only do this if you want Discord login, “Subscribe” buttons, or visit logging.

1. Go to [Supabase](https://supabase.com), sign in, and create a project (or use an existing one).
2. In the project: **Project Settings** (gear icon) → **API**. Copy:
   - **Project URL**
   - **anon public** key
3. In Cursor, open **app.js**. Near the top, find the lines with `SUPABASE_URL` and `SUPABASE_ANON_KEY`. Replace the placeholder values with your URL and anon key. Save the file.
4. In Supabase: open **SQL Editor** → **New query**. Open the file **supabase-setup.sql** from this project, copy all its contents, paste into the query box, and click **Run**. You should see a success message. That creates the tables (visits, profiles, subscriptions, etc.).

### Step 5 (optional): Discord login

Only if you want “Sign in with Discord” and Subscribe:

1. In [Discord Developer Portal](https://discord.com/developers/applications), create an application (or use an existing one). Under **OAuth2 → Redirects**, add:  
   `https://YOUR_SUPABASE_PROJECT_REF.supabase.co/auth/v1/callback`  
   (replace `YOUR_SUPABASE_PROJECT_REF` with the ref from your Supabase URL, e.g. `yajktqxbxytbnxzavago`.)
2. In Supabase: **Authentication → Providers → Discord** → turn it on, paste **Client ID** and **Client Secret** from the Discord app, then Save.

## Run it (summary)

Because this site fetches data from the internet, you should open it through a local web server (not `file://`).

**Recommended for local development (avoids 404s and CORS errors):**

```bash
node server.js
```

Then open **http://localhost:3000**. This runs a small dev server that serves the site and the `/api/steam-news` endpoint locally, so Steam updates load without 404 or proxy/CORS issues.

Other options:

- **Cursor / VS Code “Live Server”** or **`npx serve .`**: works for static files, but `/api/steam-news` won’t exist, so the app will fall back to a public CORS proxy (which may fail or be blocked). You’ll see fewer errors if the API is missing (the app tries the API once per load, then uses the proxy for the rest).

## Connect to Supabase

The site can talk directly to a Supabase project from the browser using the public (anon) key.

1. In the Supabase dashboard, create a project.
2. Go to **Project Settings → API** and copy:
   - `Project URL`
   - `anon public` key
3. In `app.js`, set:

```js
const SUPABASE_URL = "https://YOUR_PROJECT_ID.supabase.co";
const SUPABASE_ANON_KEY = "YOUR_SUPABASE_ANON_KEY";
```

4. **One-time setup (fixes 404 errors):** In Supabase go to **SQL Editor** → **New query**, open the file `supabase-setup.sql` from this repo, paste its contents, and click **Run**. That creates all required tables (visits, profiles, subscriptions, game_updates_cache). If you prefer to run statements yourself, use the snippets below.

Alternatively, create only the visits table by hand:

```sql
create table if not exists public.visits (
  id bigint generated by default as identity primary key,
  path text,
  created_at timestamptz default now()
);

alter table public.visits enable row level security;

create policy "allow inserts from anon"
  on public.visits
  for insert
  to anon
  with check (true);
```

Every time the page loads and data is fetched, the site will `insert` one row into `public.visits` (best-effort; failures are ignored in the UI). If you get a **403 on `visits`** in the console, re-run the SQL above and ensure the RLS policy `"allow inserts from anon"` exists for `public.visits`.

## Discord login (profiles)

Users can sign in with Discord to create a profile. Supabase Auth handles the OAuth flow.

### 1. Discord application

1. Go to [Discord Developer Portal](https://discord.com/developers/applications) → **New Application** (or use an existing one).
2. Open **OAuth2** → **General**. Copy the **Client ID**.
3. Under **OAuth2** → **Redirects**, add this exact redirect URL (replace `YOUR_PROJECT_REF` with your Supabase project ref, e.g. `yajktqxbxytbnxzavago`):
   ```
   https://YOUR_PROJECT_REF.supabase.co/auth/v1/callback
   ```
4. Under **OAuth2** → **General**, reset or copy the **Client Secret** (you’ll paste it into Supabase next).

### 2. Supabase: enable Discord provider

1. In Supabase: **Authentication** → **Providers** → **Discord** → enable it.
2. Paste the **Client ID** and **Client Secret** from the Discord app.
3. Save.

### 3. Profiles table (Supabase SQL)

Run this in the Supabase **SQL Editor** so signed-in users get a profile row:

```sql
create table if not exists public.profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  username text,
  avatar_url text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

alter table public.profiles enable row level security;

create policy "Users can read own profile"
  on public.profiles for select
  using (auth.uid() = id);

create policy "Users can insert own profile"
  on public.profiles for insert
  with check (auth.uid() = id);

create policy "Users can update own profile"
  on public.profiles for update
  using (auth.uid() = id);
```

After a user signs in with Discord, the app will upsert their profile (username and avatar from Discord).

**Add Discord ID to profiles** (for DM notifications). Run once:

```sql
alter table public.profiles add column if not exists discord_id text;
```

**Subscriptions table** (so users can subscribe to games and get DMs):

```sql
create table if not exists public.subscriptions (
  id bigint generated by default as identity primary key,
  user_id uuid not null references auth.users(id) on delete cascade,
  game_id text not null,
  created_at timestamptz default now(),
  unique(user_id, game_id)
);

alter table public.subscriptions enable row level security;

create policy "Users can read own subscriptions"
  on public.subscriptions for select using (auth.uid() = user_id);

create policy "Users can insert own subscriptions"
  on public.subscriptions for insert with check (auth.uid() = user_id);

create policy "Users can delete own subscriptions"
  on public.subscriptions for delete using (auth.uid() = user_id);
```

**Game updates cache** (used by the notify cron to know when a game has a new update):

```sql
create table if not exists public.game_updates_cache (
  game_id text primary key,
  last_title text,
  last_url text,
  last_date timestamptz,
  updated_at timestamptz default now()
);

alter table public.game_updates_cache enable row level security;
```

(No RLS policies = only the Supabase **service_role** key can read/write this table. Use service_role in the notify cron.)

### 4. Site URL for redirects

In Supabase: **Authentication** → **URL Configuration**. Set **Site URL** to your app’s URL (e.g. `http://localhost:3000` for local dev or your Vercel URL). Add the same URL under **Redirect URLs** if you use a strict allowlist.

## Discord DM notifications (subscribe to games)

Users who are signed in with Discord can click **Subscribe** on a game. When that game has a new update, a backend job sends them a Discord DM with a link to the site and the latest patch notes.

### 1. Discord bot (for sending DMs)

1. In the [Discord Developer Portal](https://discord.com/developers/applications), open your application (or create one).
2. Go to **Bot** → **Add Bot** → copy the **Token** (keep it secret).
3. Enable **MESSAGE CONTENT INTENT** under Bot if needed.
4. **Invite the bot to a server**: OAuth2 → URL Generator → scopes: `bot`; bot permissions: **Send Messages**, **Create Direct Messages**. Open the generated URL and add the bot to a server.
5. **Important:** Discord only allows a bot to DM a user if they share a server. So users who want DMs must be in a server where this bot is (e.g. your community server).

### 2. Vercel environment variables

For the notify cron (Vercel → Project → Settings → Environment Variables), add:

- `SUPABASE_SERVICE_ROLE_KEY` – from Supabase **Project Settings → API** (service_role, secret).
- `DISCORD_BOT_TOKEN` – your bot token from the Discord Developer Portal.
- `SITE_URL` – your site’s public URL (e.g. `https://your-app.vercel.app`) so DMs can link to it.

### 3. Cron job (Vercel)

The repo includes `api/notify-subscribers.js`, which checks for new game updates and DMs subscribers. On Vercel, add a cron in **Settings → Cron Jobs** (or use `vercel.json`):

- Path: `/api/notify-subscribers`
- Schedule: e.g. every 6 hours (`0 */6 * * *`) or daily (`0 12 * * *`).

You can also call `GET https://your-app.vercel.app/api/notify-subscribers` manually to run the job once.

**Adding new games:** When you add a game to `sources.js`, add the same game (id, name, steamAppId) to the `GAMES` array in `api/notify-subscribers.js` so the cron can notify subscribers for that game.

## Deploy on Vercel

Your setup is ready for Vercel: static files (HTML/CSS/JS) are served from the project root, and the `api/` folder runs as serverless functions (e.g. `/api/steam-news`). No build step.

### Option A: Deploy with Vercel CLI (no GitHub)

1. **Install Vercel CLI** (one time):
   ```bash
   npm install -g vercel
   ```
2. **In your project folder**, run:
   ```bash
   cd "C:\Users\Joshua\Desktop\this could be dream"
   vercel
   ```
3. **Follow the prompts:**
   - Log in or create a Vercel account if asked.
   - **Set up and deploy?** Yes.
   - **Which scope?** Your account.
   - **Link to existing project?** No (first time) or Yes if you already have one.
   - **Project name?** Accept `this-could-be-dream` or type e.g. `gamepulse`.
   - **Directory?** `./` (press Enter).
4. When it finishes, you’ll get a URL like **https://this-could-be-dream-xxx.vercel.app**. Open it to see your site.
5. **Later:** run `vercel --prod` in the same folder to deploy to production.

### Option B: Deploy with GitHub + Vercel dashboard

1. **Put the project in a GitHub repo:**
   - Create a new repo on [github.com](https://github.com/new).
   - In your project folder, run:
     ```bash
     git init
     git add .
     git commit -m "GamePulse initial"
     git branch -M main
     git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
     git push -u origin main
     ```
2. **Import in Vercel:**
   - Go to [vercel.com](https://vercel.com) and sign in (use GitHub).
   - Click **Add New… → Project**.
   - Import your GitHub repo (e.g. `YOUR_USERNAME/YOUR_REPO`).
   - **Framework Preset:** Other.
   - **Build Command:** leave blank.
   - **Output Directory:** leave as `.` or blank.
   - Click **Deploy**.
3. Vercel will build and give you a URL like **https://your-repo.vercel.app**. Every push to `main` will auto-deploy.

### After deploy

- **Supabase:** In Supabase **Authentication → URL Configuration**, set **Site URL** to your Vercel URL (e.g. `https://your-app.vercel.app`) and add it to **Redirect URLs**.
- **Discord / cron:** If you use Discord login and the notify cron, add env vars in Vercel (**Project → Settings → Environment Variables**): `SUPABASE_SERVICE_ROLE_KEY`, `DISCORD_BOT_TOKEN`, `SITE_URL`. The cron for `/api/notify-subscribers` is already in `vercel.json`.

## How it fetches data

**When deployed on Vercel:** The site calls `/api/steam-news?appid=...` (same origin), so Steam news is fetched server-side with no CORS. That keeps game cards **current** and avoids “Unavailable” from blocked proxies.

**When running locally (or if the API isn’t available):** The app falls back to public CORS proxies (allorigins.win, then corsproxy.io). If a source blocks or rate-limits the proxy, that card may show “Unavailable”. The app also retries once after a short delay to reduce transient failures.

### Swap or add proxies

In `app.js` the list `CORS_PROXIES` is tried in order. You can add or change entries (e.g. your own proxy URL) so the app has more fallbacks when running without the Vercel API.

## Customize: add/remove games

Edit `sources.js`.

Each game uses Steam’s public app news feed:

```js
{
  id: "dota2",
  name: "Dota 2",
  steamAppId: 570,
  keywords: ["patch", "update"],
  officialUrl: "https://www.dota2.com/",
}
```

- `steamAppId`: the Steam app id for the game
- `keywords`: used to choose the “most patch-note-like” entry from the last few Steam news items
- `officialUrl`: fallback link if a feed fails

## Customize: gaming news sources

Also in `sources.js`, edit `NEWS_FEEDS`:

```js
{ id: "pcgamer", name: "PC Gamer", url: "https://www.pcgamer.com/rss/" }
```

The app supports both **RSS** and **Atom** feeds.

## Notes / limitations

- This is an MVP: “patch notes” are best-effort (Steam news is sometimes announcements, not always formal patch notes).
- The public proxy may be slow, rate-limited, or blocked by some publishers.
- Results are cached in `localStorage` for ~15 minutes to reduce repeated requests.
